主要项目经验

  • simple-hpa

    实现的功能: 基于Pod的QPS实现HPA

    在开发期间的2021年，原生Kubernetes的HPA只能做到基于CPU与内存的自动扩展能力，但实际业务场景更多是基于服务的QPS来决定最高上限。因官方扩展QPS来达到自动扩展的能力的方式个人认为太重，因此开发。主要实现的功能是根据NGINX Ingress的访问日志来计算服务的单个Pod的平均QPS,并实现基于QPS的HPA

    技术栈:

    • Golang
    • Kubernetes
    • NGINX

    主要任务：

    1. 代码实现
2. 架构设计
3. 测试

    项目地址: https://github.com/UnknowName/simple-hpa

  • Monitor-auto

    实现的功能: HTTP健康检测，并尝试自动恢复

    前期项目应用运行环境为Windows + IIS方式，期间发现应用不定期发生假死现象，于时开发此应用用于自动干预恢复，减少人工。

    1. 异步检测，最快10秒内响应
2. 异常自动从网关中下线主机并进行自动恢复(先回收，回收失败后重启)
3. 恢复后自动加回网关，继续对外提供服务
4. 异常/恢复发送通知。支持微信/钉钉/邮件

    涉及到技术栈与库:

    • Python
    • aiohttp
    • ansible
    • PowerShell

    项目地址: https://github.com/UnknowName/monitor-auto
