主要项目经验

  • simple-hpa

    在开发期间的2021年，原生Kubernetes的HPA只能做到基于CPU与内存的自动扩展能力，但实际业务场景更多是基于服务的QPS来决定最高上限。因官方扩展QPS来达到自动扩展的能力的方式个人认为太重，因此开发。

    实现的功能：

    1. 通过将Ingress NGINX的日志转发至应用
2. 应用计算指定需要自动伸缩的应用的`QPS`
3. 根据指定应用的`QPS`承载能力，伸缩至最佳数量的副本

    技术栈:

    • Golang
    • Kubernetes
    • NGINX

    主要任务：

    1. 代码实现
2. 架构设计
3. 测试

    项目地址: https://github.com/UnknowName/simple-hpa

  • Monitor-auto

    实现的功能: HTTP健康检测，并尝试自动恢复

    前期项目应用运行环境为Windows + IIS方式，期间发现应用不定期发生假死现象，于时开发此应用用于自动干预恢复，减少人工。

    1. 异步检测，最快10秒内响应
2. 异常自动从网关中下线主机并进行自动恢复(先回收，回收失败后重启)
3. 恢复后自动加回网关，继续对外提供服务
4. 异常/恢复发送通知。支持微信/钉钉/邮件

    涉及到技术栈与库:

    • Python
    • aiohttp
    • ansible
    • PowerShell

    项目地址: https://github.com/UnknowName/monitor-auto

  • 基础服务一键部署

    各项目基础架构相同，中间件也使用相同。各项目之间有着共同性。为推进工作效率与标准化安装流程，将基础服务Mongo/MySQL/Kafka/RabbitMQ/Hazeclast/Elasticsearch/HAProxy/FastDFS实现使用Ansible部署集群。
