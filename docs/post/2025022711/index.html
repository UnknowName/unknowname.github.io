主要项目经验

1. simple-hpa

项目描述： 在2021年，为提升Kubernetes的自动扩展能力，开发了一款基于QPS（每秒查询数）的Horizontal Pod Autoscaler，解决原生Kubernetes只能基于CPU和内存进行扩展的局限性。

  • 技术栈: Golang, Kubernetes
  • 主要功能:
    • 日志转发： 将Ingress NGINX的访问日志实时转发至应用，提供必要的监控数据
    • QPS计算：计算指定服务的QPS，以实时评估服务的负载需求。
    • 动态伸缩： 根据计算得出的QPS承载能力，自动调整Kubernetes中应用的副本数量，确保最佳性能和资源利用。
  • 职责:
    • 实现核心代码，确保能够精准计算并响应服务负载变化。
    • 设计系统架构，确保高可用性和可扩展性，以支持大型环境下的自动伸缩需求。
    • 制定并执行全面的测试计划，验证功能的可靠性和性能。
  • 成就:
    • 成功实现了基于QPS的自动扩展功能，显著提升了特定服务在高负载下的处理能力。
    • 项目上线后，系统在高峰时段稳定性提升了50%，并有效降低了资源浪费。

2. Monitor-auto

项目描述： 用于HTTP健康检测和自动恢复的工具，旨在减少因应用假死带来的人工干预，提高系统稳定性。

  • 技术栈： Python, aiohttp, Ansible, PowerShell
  • 主要功能:
    • 异步健康检测： 实现每10秒内快速响应的异步检测机制，及时发现服务异常。
    • 自动异常处理： 当检测到异常时，自动将主机下线，并通过回收和重启策略进行恢复，确保最小化服务中断时间。
    • 服务恢复： 恢复后，系统会自动将主机重新加入网关，继续对外提供无缝服务。
    • 通知机制： 在发生异常或恢复时，通过微信、钉钉和邮件等多种方式发送通知，确保相关人员及时获知。
  • 职责:
    • 设计并实现HTTP健康检测功能，提升了应用监控的实时性和准确度。
    • 利用 aiohttp 库构建异步请求，显著减少响应时间，并增强系统并发处理能力。
    • 集成 Ansible 和 PowerShell 脚本，实现自动化运维，简化了恢复流程。
    • 开发并维护通知系统，确保团队及时收到关键事件信息。
  • 成就：
    • 成功上线后，应用的自动恢复能力将人工干预时间减少了90%。
    • 系统监控覆盖率达到了100%，提高了整体服务的稳定性和用户满意度。
